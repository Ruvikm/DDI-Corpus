# DDI-Corpus

> 数据中心部署意图语料库

DDI-Corpus面向数据中心服务功能链部署场景，主要整理高层业务需求与网络部署维度之间的对应关系。其样本贴近客户提出业务目标，工程人员补齐部署要求的运维语境，重点保留稳定性、安全边界等信息。

本数据集出自正在向《软件学报》投稿的论文《基于Agent Skills的标准驱动服务功能链意图实现的关键技术》。当前仓库暂时提供部分数据集预览，论文录用时将公开完整数据集。

## 📁 数据文件

| 文件名 | 内容 |
| --- | --- |
| `DDI-Corpus_preview_top50.xlsx` | 部分数据集预览，共50条样本 |

完整数据集围绕真实业务系统实体构建。其系统名称取自公开官方信息，样本覆盖政务、金融、企业管理、数据库、中间件、DevOps、安全和通用组件等类别。语料内容面向数据中心部署需求，涉及防火墙访问控制、负载均衡、网关接入、流量整形、缓存加速和入侵防御等技术部署维度。

## 🔎 数据示例

以下选取5条预览样本，示例中保留行业、系统名称、技术维度和需求句子。

### 示例1

**行业**：政务

**系统名称**：广西政务服务（广西数字政务一体化平台）

**技术维度**：防火墙访问控制；网络监测与镜像采集；DPI深度检测；缓存加速；IDS和IPS入侵防御

**需求句子**：在数据中心部署广西政务服务（广西数字政务一体化平台）系统，要求广西政务服务（广西数字政务一体化平台）面向对外访问使用场景，首要目标是稳定可用。遇到突发访问或异常流量时，保证核心功能可用并可快速恢复。防火墙对外接口按白名单与鉴权策略控制访问，未授权来源默认拒绝。常用页面和静态资源建议内容缓存，降低中心负载。配套完善的网络监控、告警与审计日志，方便运维排障和追溯。

### 示例2

**行业**：企业管理

**系统名称**：致远互联M3

**技术维度**：防火墙访问控制；网络监测与镜像采集；CDN边缘盒子；缓存加速；IDS和IPS入侵防御

**需求句子**：在数据中心部署致远互联M3系统，要求致远互联M3承担协同办公类业务访问，需把可用性、性能和安全控制放在前面。遇到突发访问或异常流量时，保证核心功能可用并可快速恢复。防火墙访问控制策略要清晰，关键接口建议白名单与鉴权，默认拒绝未授权请求。对热点内容可做内容缓存或内容分发节点，减少回源压力并提升响应。监控指标和访问日志要齐全，出现异常能快速定位并回溯。

### 示例3

**行业**：数据库

**系统名称**：Redis

**技术维度**：防火墙访问控制；网络监测与镜像采集；会话控制和连接限速；流量整形与QoS；IPSec隧道加密；密码机和加密设备接入

**需求句子**：在数据中心部署Redis系统，要求在对外访问场景下，Redis需要保证高峰期访问仍然顺畅。遇到突发访问或异常流量时，保证核心功能可用并可快速恢复。防火墙对外接口按白名单与鉴权策略控制访问，未授权来源默认拒绝。必要时支持流量整形与会话控制，保障关键业务在压力下仍能正常办理。对敏感数据建议全程加密处理，满足合规要求。
